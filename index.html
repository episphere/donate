epiDonate [code]

Problem - Many cohort studies require mechanisms for data donation. Typical examples are the donation of consumer genomics and wearable sensor data.

It is however not reasonable to expect participants to have the computational skills, or the patience, to handle the data extraction and submission on their own. Instead, the data donnation should be integrated in the cohort Web Application such that the data can be retireved by engaging the consumer-facing APIs on behalf of the participant.

Solution - A web service was developed to manage individual invitation tokens for data donation. This way, the donation is reduced to clicking on a donation button in the same App that acquires the data on behalf of the devide owner.

The design of the web service explores what could be a Minimal infrastructure for data donation using the modern Web Stack. The API only needs two commands to handle the transactions, get and post, matching the HTTP GET and POST methods. The service automatically creates user and admin tokens stored in a gitignored data folder automatically created to hold them.
